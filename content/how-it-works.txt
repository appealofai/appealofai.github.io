APPEALOFAI CONTENT

Short version:

1. Write and verify the article.
2. Create the public HTML page in articles/.
3. Add the public metadata to content/articles.json.
4. Put the X companion copy in content/drafts/article-id.x.txt if useful.

Automatic behavior:

- The Notes ticker reads content/articles.json.
- Only published items appear in the ticker.
- Draft text files are not published automatically.

Why this is the current best setup:

- It is static, simple and safe.
- Every public page can be reviewed before it goes live.
- The same article metadata can power the front page, archive and ticker.
- Automation can be added later if publishing becomes repetitive.

Current menu:

- Journal
- Articles
- About

Do not add menu items until there is enough real content for them.
